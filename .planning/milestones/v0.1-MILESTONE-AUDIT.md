### Gap 1 — GRAPH-04: CommitRow missing merge-commit visual (BLOCKER)

| | Detail |
|--|--|
| **Affected requirement** | GRAPH-04 |
| **Rust side** | `GraphCommit.is_merge: bool` computed correctly in `graph.rs`; serialized in DTO |
| **TypeScript side** | `GraphCommit` interface in `types.ts` includes `is_merge: boolean` |
| **Frontend gap** | `CommitRow.svelte` receives the full `commit` prop but never reads `commit.is_merge` — no larger dot, no ring stroke for merge commits |
| **Fix** | Add `commit.is_merge` conditional to `LaneSvg.svelte` or `CommitRow.svelte` dot rendering |

### Gap 2 — Checkout/create-branch does not deterministically refresh StagingPanel (Non-blocker)

| | Detail |
|--|--|
| **Affected requirements** | BRNCH-03 (checkout), STAGE-04 (auto-refresh) interaction |
| **Root cause** | `checkout_branch` and `create_branch` in `commands/branches.rs` do not accept `AppHandle` and never call `app.emit("repo-changed", ...)` |
| **Current behavior** | StagingPanel refreshes only if the FSEvents watcher fires (300ms debounce after real filesystem changes). For a checkout to an identical-tree branch, watcher fires 0 events → StagingPanel is stale |
| **Reliable path** | CommitGraph remount works correctly via `onrefreshed → handleRefresh() → graphKey++` |
| **Fix** | Add `app: AppHandle` parameter to both commands; emit `repo-changed` after the git op, mirroring the pattern in `commands/commit.rs` lines 103–118 |
| **Severity** | Non-blocking for current requirements (STAGE-04 requirement text covers external tool changes). Behavioral gap for checkout flow consistency. |

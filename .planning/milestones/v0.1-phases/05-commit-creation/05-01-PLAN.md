<feature>
  <name>Commit creation and amend commands</name>
  <files>src-tauri/src/commands/commit.rs, src-tauri/src/git/types.rs</files>
  <behavior>
    Tests to write FIRST (RED phase):

    - create_commit_creates_commit: Given a repo with a staged file, create_commit_inner creates a commit; HEAD now points to a commit with the expected summary.
    - create_commit_unborn_head: Given a freshly init'd repo (no commits, unborn HEAD) with a staged file, create_commit_inner succeeds (does not panic or error on missing parents).
    - create_commit_uses_signature: Given a repo with gitconfig user.name/email set, the resulting commit's author name matches the configured value. (Use git2 Repository::config() + set_str() to set user.name and user.email in the test repo before calling inner fn.)
    - amend_commit_updates_message: Given a repo with an existing commit, amend_commit_inner with a new subject updates the HEAD commit's summary to the new subject.
    - amend_commit_includes_staged: Given a repo with an existing commit and a new staged file, amend_commit_inner creates an amended commit whose tree includes the newly staged file.
    - get_head_commit_message_returns_message: Given a repo with a commit whose message has a subject and body ("Subject\n\nBody text"), get_head_commit_message_inner returns HeadCommitMessage { subject: "Subject", body: Some("Body text") }.
  </behavior>
  <implementation>
    Step 1 — Add HeadCommitMessage to src-tauri/src/git/types.rs:
    ```rust
    #[derive(Debug, Clone, Serialize, Deserialize)]
    pub struct HeadCommitMessage {
        pub subject: String,
        pub body: Option<String>,
    }
    ```

    Step 2 — Implement inner fns in src-tauri/src/commands/commit.rs:

    create_commit_inner(path, subject, body, state_map):
    - open_repo_from_state(path, state_map)
    - repo.signature()
    - repo.index() + index.write_tree() + repo.find_tree()
    - Build message: if body is Some and non-empty-trimmed -> "{subject}\n\n{body}", else subject
    - Build parents vec using unborn HEAD detection pattern (ErrorCode::UnbornBranch -> empty vec)
    - repo.commit(Some("HEAD"), &sig, &sig, &message, &tree, &parent_refs)

    amend_commit_inner(path, subject, body, state_map):
    - open_repo_from_state(path, state_map)
    - let head_commit = repo.head()?.peel_to_commit()?
    - repo.signature()
    - repo.index() + index.write_tree() + repo.find_tree() — always include current staged tree
    - Build message same as create_commit_inner
    - head_commit.amend(Some("HEAD"), Some(&sig), Some(&sig), None, Some(&message), Some(&tree))

    get_head_commit_message_inner(path, state_map):
    - open_repo_from_state(path, state_map)
    - let commit = repo.head()?.peel_to_commit()?
    - HeadCommitMessage { subject: commit.summary().unwrap_or("").to_owned(), body: commit.body().map(str::to_owned) }

    Step 3 — Tauri wrapper fns (#[tauri::command] async):
    - create_commit: clones state_map, calls spawn_blocking with create_commit_inner, then cache.0.lock().unwrap().remove(&path), then app.emit("repo-changed", path)
    - amend_commit: same pattern as create_commit (cache invalidate + emit)
    - get_head_commit_message: read-only, no cache invalidation or event emission needed; returns Result<HeadCommitMessage, String>

    Note: Do NOT register commands in lib.rs yet — that is Plan 03's task.
  </implementation>
</feature>

  <behavior>
    - Submitting with empty subject sets subjectError = 'Subject is required'; no invoke fires
    - Submitting with empty staged files (non-amend, stagedCount === 0) sets stagedError = 'No files staged'; no invoke fires
    - Submitting in amend mode with empty staged area succeeds (no stagedError — amend allows empty staging)
    - Editing subject input after a subjectError clears subjectError (oninput handler)
    - Checking amend checkbox calls get_head_commit_message and populates subject/body fields
    - Unchecking amend checkbox reverts subject and body to empty
    - During invoke, committing = true (button shows loading state, disabled)
    - On successful create_commit invoke: subject = '', body = '', amend = false, committing = false
    - On successful amend_commit invoke: same reset
  </behavior>
  <action>
    Add HeadCommitMessage interface to src/lib/types.ts:
    ```typescript
    export interface HeadCommitMessage {
      subject: string;
      body: string | null;
    }
    ```

    Create src/components/CommitForm.svelte with Svelte 5 runes:

    Props: { repoPath: string; stagedCount: number }

    State:
    - subject = $state('')
    - body = $state('')
    - amend = $state(false)
    - committing = $state(false)
    - subjectError = $state('')
    - stagedError = $state('')

    handleSubmit():
    1. Clear both errors
    2. If !subject.trim() -> subjectError = 'Subject is required'; return
    3. If !amend && stagedCount === 0 -> stagedError = 'No files staged'; return
    4. committing = true
    5. try: if amend -> safeInvoke('amend_commit', { path: repoPath, subject: subject.trim(), body: body.trim() || null })
           else -> safeInvoke('create_commit', { path: repoPath, subject: subject.trim(), body: body.trim() || null })
    6. On success: subject = ''; body = ''; amend = false
    7. catch: surface error (log to console; no toast in v0.1)
    8. finally: committing = false

    handleAmendToggle(checked: boolean):
    - If checked: fetch head message via safeInvoke<HeadCommitMessage>('get_head_commit_message', { path: repoPath })
      and set subject = msg.subject; body = msg.body ?? ''
    - If unchecked: subject = ''; body = ''

    Also clear stagedError in $effect when stagedCount changes or amend changes.

    Layout (inside 240px column, no extra width):
    - Outer div: padding 8px, display flex, flex-direction column, gap 6px, flex-shrink 0
    - Subject: single-line input (type="text"), full width, placeholder="Summary (required)"
      - oninput: if subjectError -> subjectError = ''
      - border: 1px solid var(--color-border), background var(--color-surface), color var(--color-text)
      - border-radius 4px, padding 4px 6px, font-size 12px
    - subjectError: if shown, small text below subject in red (#f87171 or similar warm red)
    - Body: textarea, rows=3, full width, placeholder="Description (optional)"
      - Same styling as subject input; resize: vertical
    - Amend checkbox row: display flex, align-items center, gap 6px
      - input type=checkbox bound to amend, oninput fires handleAmendToggle
      - label text: "Amend previous commit", font-size 12px, color var(--color-text-muted)
    - stagedError: if shown, small text in red below amend row
    - Commit button: full width, height 28px, background var(--color-accent), color white,
      border-radius 4px, font-size 12px, cursor pointer
      - disabled={committing}; opacity 0.6 when disabled
      - Text: committing ? 'Committing...' : amend ? 'Amend Commit' : 'Commit'
  </action>

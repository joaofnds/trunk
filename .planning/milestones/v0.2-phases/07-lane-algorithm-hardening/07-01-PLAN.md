    **Fix 1 -- Ghost lanes (ALGO-01):**
    The pass-through edge emission at line 80-92 iterates `active_lanes` and emits Straight edges for all occupied slots except the current commit's column. The bug is that after a merge, when the first parent is already claimed at a different column (the ForkLeft/ForkRight case at line 104-125), the current column stays `None` (correct -- lane terminates). However, if the merged branch's OID lingers in `active_lanes` from `pending_parents` pre-population, it may still appear occupied. Fix: After consuming the commit's slot (`active_lanes[col] = None` at line 96), also verify that no other `active_lanes` entry contains an OID that is no longer in `pending_parents` (stale entry). Pragmatically: the current code already sets `active_lanes[col] = None` at line 96. The ghost lane issue is more likely from the pass-through edges being emitted BEFORE the slot is cleared. The order is already correct (pass-through at 80-92, then clear at 96), so the pass-through correctly skips the commit's own column (`other_col != col`). The actual ghost scenario: branch A merges into main, A's column becomes None, but A's parent was in head_chain and thus in pending_parents -- the parent's column is still occupied. This is correct behavior (the parent is still active). The real ghost is when `active_lanes` has a `Some(oid)` where that oid was already processed. Fix: after processing each commit, if the commit has no parents (root), ensure its slot is None. If the commit's first parent redirects to another column (merge case), ensure the original column is None and stays None.

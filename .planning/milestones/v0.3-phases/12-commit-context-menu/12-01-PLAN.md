  <name>Task 1: Create commit_actions.rs with checkout_commit, create_tag, cherry_pick, revert_commit</name>
  <files>src-tauri/src/commands/commit_actions.rs, src-tauri/src/commands/mod.rs</files>
  <behavior>
    - checkout_commit_inner: given a valid OID on a clean workdir, detaches HEAD at that OID and returns GraphResult with re-walked commits
    - checkout_commit_inner: given a dirty workdir, returns TrunkError with code "dirty_workdir" without modifying HEAD
    - create_tag_inner: given valid OID + tag_name + message, creates annotated tag using repo.signature() and returns GraphResult
    - create_tag_inner: given empty message, uses tag_name as message
    - create_tag_inner: given duplicate tag name, returns TrunkError (git_error)
    - cherry_pick_inner: given a valid non-merge commit OID, runs `git cherry-pick <oid>` with GIT_TERMINAL_PROMPT=0 and returns GraphResult
    - cherry_pick_inner: when git cherry-pick exits non-zero with "conflict" in stderr, returns TrunkError with code "conflict_state"
    - revert_commit_inner: given a valid non-merge commit OID, runs `git revert <oid> --no-edit` with GIT_TERMINAL_PROMPT=0 and returns GraphResult
    - revert_commit_inner: when git revert exits non-zero with "conflict" in stderr, returns TrunkError with code "conflict_state"
  </behavior>
  <action>
Create `src-tauri/src/commands/commit_actions.rs` following the established stash.rs pattern:

1. **Module setup**: Import HashMap, PathBuf, State, AppHandle, Emitter, TrunkError, graph, types::{GraphResult}, state::{CommitCache, RepoState}.

2. **Helper**: `fn open_repo(path, state_map) -> Result<Repository, TrunkError>` -- same as stash.rs pattern. Also `fn is_dirty(repo) -> Result<bool, git2::Error>` -- copy from branches.rs (or make it pub there and import; simpler to duplicate the small fn to avoid circular deps).

3. **checkout_commit_inner(path, oid, state_map) -> Result<GraphResult, TrunkError>**:
   - Open repo, check is_dirty -> return dirty_workdir error
   - `repo.revparse_single(oid)` to get object
   - `repo.checkout_tree(&obj, Some(CheckoutBuilder::new().safe()))` then `repo.set_head_detached(obj.id())`
   - Drop repo, reopen, `graph::walk_commits(&mut repo2, 0, usize::MAX)`

4. **create_tag_inner(path, oid, tag_name, message, state_map) -> Result<GraphResult, TrunkError>**:
   - Open repo, `revparse_single(oid)`, `repo.signature()`
   - If message.trim().is_empty(), use tag_name as message
   - `repo.tag(tag_name, &obj, &sig, msg, false)` (false = no force)
   - Drop repo, reopen, walk_commits

5. **cherry_pick_inner(path, oid, state_map) -> Result<GraphResult, TrunkError>**:
   - Resolve path_buf from state_map
   - `std::process::Command::new("git").args(["cherry-pick", oid]).current_dir(path_buf).env("GIT_TERMINAL_PROMPT", "0").output()`
   - If !output.status.success(): check stderr for "conflict" -> conflict_state error, else cherry_pick_error
   - Open repo, walk_commits

6. **revert_commit_inner(path, oid, state_map) -> Result<GraphResult, TrunkError>**:
   - Same pattern as cherry_pick but args `["revert", oid, "--no-edit"]`
   - Error codes: "conflict_state" or "revert_error"

7. **Outer #[tauri::command] wrappers** for each: follow stash.rs pattern exactly -- spawn_blocking, update CommitCache, emit "repo-changed" via AppHandle.

8. Add `pub mod commit_actions;` to `src-tauri/src/commands/mod.rs`.

Key details:
- All git CLI commands must set `GIT_TERMINAL_PROMPT=0` env var (per v0.3 decision)
- Revert MUST pass `--no-edit` to prevent editor blocking (Pitfall 7)
- Drop first repo before reopening for graph walk (Pitfall 2)
- Use `repo.signature()` for annotated tags (Pitfall 4)

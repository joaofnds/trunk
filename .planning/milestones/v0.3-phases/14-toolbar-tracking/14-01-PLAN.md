In list_refs_inner, replace the hardcoded `ahead: 0, behind: 0` in the local branch iteration with real values from `repo.graph_ahead_behind()`.

Implementation:
1. Inside the `.map(|(branch, _)| { ... })` closure for local branches, after resolving `upstream` and `last_commit_timestamp`:
2. Extract `local_oid` from `branch.get().target()` FIRST (before calling upstream again).
3. Compute ahead/behind only when upstream is `Some` and `local_oid` is `Some`:
   ```rust
   let (ahead, behind) = match (&upstream, branch.get().target()) {
       (Some(_), Some(local_oid)) => {
           branch.upstream()
               .ok()
               .and_then(|ub| ub.get().target())
               .map(|remote_oid| repo.graph_ahead_behind(local_oid, remote_oid).unwrap_or((0, 0)))
               .unwrap_or((0, 0))
       }
       _ => (0, 0),
   };
   ```
4. Use the computed `ahead` and `behind` in the BranchInfo constructor instead of 0.

Pitfall avoidance: Extract local_oid via `branch.get().target()` before calling `branch.upstream()` to avoid double-borrow issues. Use `unwrap_or((0, 0))` to handle errors gracefully (unborn branches, missing upstream refs).

Also add a test `list_refs_ahead_behind` that creates a test repo with a local branch ahead of its tracking branch and verifies non-zero ahead count.

    - Test: In a linear 3-commit repo (no merges), every non-root commit (i.e., every commit except the last in topo order) has at least one Straight edge where from_column == to_column == commit.column
    - Test: The root commit (no parents) has zero Straight edges originating from its own column (only pass-through edges from other lanes, if any)
    - Test: In a merge repo, the merge commit has both a Straight first-parent edge AND a MergeLeft/MergeRight secondary-parent edge
  </behavior>
  <action>
    **Bug location:** `src-tauri/src/git/graph.rs`, inside the `walk_commits` function, in the first-parent handling block (the `if idx == 0` branch inside the parent loop, approximately lines 75-85).

    **Root cause:** The first-parent block does lane bookkeeping (`active_lanes[col] = Some(parent_oid)` and `pending_parents.insert(parent_oid, col)`) but never pushes a `GraphEdge` onto the `edges` vector. Only the secondary-parent branch (the `else` block) and the pass-through loop emit edges.

    **Fix:** After the first-parent lane assignment succeeds (i.e., after `active_lanes[col] = Some(parent_oid); pending_parents.insert(parent_oid, col);`), add:

    ```rust
    edges.push(GraphEdge {
        from_column: col,
        to_column: col,
        edge_type: EdgeType::Straight,
        color_index: col,
    });
    ```

    This edge must also be emitted in the case where the parent is already pending from another child (the "if already in pending" case). In that scenario, the current column's lane ends (slot stays None), and we need a crossing edge from `col` to the existing pending column. Check: if `pending_parents.contains_key(&parent_oid)`, get the existing column with `pending_parents[&parent_oid]` and emit an edge from `col` to that column. If same column, it is Straight. If different columns, determine ForkLeft/ForkRight based on column comparison.

    **Important:** Only emit the first-parent edge when the commit actually HAS parents (i.e., `parents.len() > 0`). Root commits have zero parents and should not get a first-parent edge.

    **Test changes (RED first, then GREEN):**

    1. In the existing `linear_topology` test, add an assertion BEFORE the fix (it will fail, confirming the bug):
       ```rust
       // Every non-root commit must have a Straight edge at its own column
       for c in &commits[..commits.len()-1] {
           let has_own_straight = c.edges.iter().any(|e| {
               matches!(e.edge_type, EdgeType::Straight)
                   && e.from_column == c.column
                   && e.to_column == c.column
           });
           assert!(has_own_straight, "commit {} missing first-parent Straight edge", c.short_oid);
       }
       // Root commit should NOT have a self-straight edge
       let root = commits.last().unwrap();
       let root_has_self_straight = root.edges.iter().any(|e| {
           matches!(e.edge_type, EdgeType::Straight) && e.from_column == root.column && e.to_column == root.column
       });
       assert!(!root_has_self_straight, "root commit should not have self-straight edge");
       ```

    2. Add a new test `merge_has_first_parent_straight`:

           });

       }

<objective>
Fix branch lane column assignment and fork edge topology in the commit graph algorithm.

Purpose: Branch lanes appear as isolated vertical segments instead of forking from the parent commit row because (1) side branches steal column 0 from HEAD/main due to revwalk order, and (2) fork edges appear on the wrong rows as a consequence. The fix pre-reserves column 0 for HEAD's first-parent chain so the main line stays visually primary and branch fork points render at the correct divergence rows.
Output: Corrected graph.rs with HEAD-priority column assignment, correct fork topology, and regression tests.

    **Root cause (from .planning/debug/branch-fork-edges-missing.md):**

    The revwalk (TOPOLOGICAL|TIME) visits branch tips before shared ancestors. Whichever branch tip is encountered first claims column 0. With no priority heuristic, side branches steal column 0 from HEAD/main, inverting the visual hierarchy. Fork edges then appear on the wrong rows because main gets pushed to higher columns.

    **Fix — two changes to walk_commits() in src-tauri/src/git/graph.rs:**

    **Change 1: Pre-compute HEAD's first-parent chain and reserve column 0.**

    Before the main walk loop (before line 34 `for &oid in &oids`), add:

    ```rust
    // Pre-reserve column 0 for HEAD's first-parent chain
    let mut head_chain: std::collections::HashSet<git2::Oid> = std::collections::HashSet::new();
    if let Ok(head_ref) = repo.head() {
        if let Some(oid) = head_ref.target() {
            let mut current = Some(oid);
            while let Some(c_oid) = current {
                head_chain.insert(c_oid);
                current = repo.find_commit(c_oid).ok().and_then(|c| c.parent_id(0).ok());
            }
        }
    }
    ```

    **Change 2: Use head_chain in column assignment.**

    Modify the column assignment block (lines 39-50). When a commit is NOT in `pending_parents` (i.e., it's a new chain needing a column), check if it belongs to `head_chain`. If it does AND column 0 is free, assign it column 0. If it does NOT belong to head_chain, skip column 0 when looking for a free lane (start search at column 1 if column 0 is free but reserved for HEAD chain).

    Replace the "New chain: find first free lane or create one" block (lines 43-49) with:

    ```rust
    // New chain: find first free lane or create one
    if head_chain.contains(&oid) {
        // HEAD chain member: prefer column 0
        if active_lanes.is_empty() || active_lanes[0].is_none() {
            if active_lanes.is_empty() {
                active_lanes.push(None);
            }
            0
        } else {
            // Column 0 occupied — fall back to first free
            if let Some(i) = active_lanes.iter().position(|s| s.is_none()) {
                i
            } else {
                active_lanes.push(None);
                active_lanes.len() - 1
            }
        }
    } else {
        // Non-HEAD chain: skip column 0 if it might be needed by HEAD chain
        let start_col = if !head_chain.is_empty() && (active_lanes.is_empty() || active_lanes[0].is_none()) {
            1 // Reserve col 0 for HEAD chain
        } else {
            0
        };
        if let Some(i) = active_lanes.iter().skip(start_col).position(|s| s.is_none()) {
            i + start_col
        } else {
            active_lanes.push(None);
            active_lanes.len() - 1
        }
    }
    ```

    **Why this fixes fork edges too (Root Cause #2):** Once HEAD/main occupies column 0, branch tips are assigned to higher columns. When a branch tip's first parent is already pending at column 0 (claimed by a HEAD-chain commit processed earlier), the existing "already pending" code path (lines 77-92) correctly emits a ForkLeft edge from the branch tip's column to column 0. The fork edge now appears on the correct row (the branch tip's row) and goes in the correct direction (branch forking from main, not main forking from branch).

        idx.add_path(std::path::Path::new("f1.txt")).unwrap();
        idx.write().unwrap();

        std::fs::write(dir.path().join("f2.txt"), "f2").unwrap();
        let mut idx = repo.index().unwrap();
        idx.add_path(std::path::Path::new("f2.txt")).unwrap();

        std::fs::write(dir.path().join("b0.txt"), "b0").unwrap();
        let mut idx = repo.index().unwrap();
        idx.add_path(std::path::Path::new("b0.txt")).unwrap();
        idx.write().unwrap();
        let tree = repo.find_tree(idx.write_tree().unwrap()).unwrap();
        let _b0 = repo.commit(Some("refs/heads/topic"), &sig, &sig, "B0", &tree, &[&c1_commit]).unwrap();

        let mut repo = git2::Repository::open(dir.path()).unwrap();
        let commits = walk_commits(&mut repo, 0, usize::MAX).unwrap();

        // Find commits by summary
        let c2 = commits.iter().find(|c| c.summary == "C2").expect("C2 not found");
        let c1 = commits.iter().find(|c| c.summary == "C1").expect("C1 not found");
        let c0 = commits.iter().find(|c| c.summary == "C0").expect("C0 not found");
        let b0 = commits.iter().find(|c| c.summary == "B0").expect("B0 not found");

        // HEAD chain (C2, C1, C0) must all be at column 0
        assert_eq!(c2.column, 0, "C2 (HEAD) should be at column 0");
        assert_eq!(c1.column, 0, "C1 should be at column 0");
        assert_eq!(c0.column, 0, "C0 should be at column 0");

        // Topic branch tip must NOT be at column 0
        assert!(b0.column > 0, "B0 (topic branch) should be at column > 0, got {}", b0.column);

        // B0 must have a fork edge toward column 0 (connecting to parent C1)
        let has_fork_to_main = b0.edges.iter().any(|e| {
            matches!(e.edge_type, EdgeType::ForkLeft) && e.to_column == 0
        });
        assert!(has_fork_to_main, "B0 should have ForkLeft edge toward column 0, edges: {:?}", b0.edges);

        // Main chain commits should NOT have ForkLeft/ForkRight edges originating from their own column
        for main_commit in [c2, c1, c0] {

                    && e.from_column == main_commit.column
            });
            assert!(!has_own_fork, "main commit {} should not have fork edges from its own column", main_commit.summary);
        }
    }

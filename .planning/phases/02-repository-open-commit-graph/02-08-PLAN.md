<objective>
Fix branch lane column assignment and fork edge topology in the commit graph algorithm.

Purpose: Branch lanes appear as isolated vertical segments instead of forking from the parent commit row because (1) side branches steal column 0 from HEAD/main due to revwalk order, and (2) fork edges appear on the wrong rows as a consequence. The fix pre-reserves column 0 for HEAD's first-parent chain so the main line stays visually primary and branch fork points render at the correct divergence rows.
Output: Corrected graph.rs with HEAD-priority column assignment, correct fork topology, and regression tests.
</objective>

  <action>
    **Root cause (from .planning/debug/branch-fork-edges-missing.md):**

    The revwalk (TOPOLOGICAL|TIME) visits branch tips before shared ancestors. With B0 (topic, newer timestamp) and C2 (main/HEAD) sharing ancestor C1, the revwalk may visit B0 FIRST. Without any priority heuristic, B0 claims column 0, AND B0 reserves its parent C1 at column 0 via pending_parents. When C2 is later processed, even if we steer C2 to column 0 based on head_chain membership, C2's first parent C1 is already reserved at column 0 by B0. This means:
    - C1 ends up at whatever column B0 reserved (not necessarily col 0 if we only fix new-chain assignment)
    - The fork edge direction is wrong (main appears to fork from branch, not branch from main)

    **Fix — three changes to walk_commits() in src-tauri/src/git/graph.rs:**

    **Change 1: Pre-compute HEAD's first-parent chain AND pre-populate pending_parents at col 0.**

    Before the main walk loop (before line 34 `for &oid in &oids`), add:

    ```rust
    // Pre-compute HEAD's first-parent chain
    let mut head_chain: std::collections::HashSet<git2::Oid> = std::collections::HashSet::new();
    if let Ok(head_ref) = repo.head() {
        if let Some(oid) = head_ref.target() {
            let mut current = Some(oid);
            while let Some(c_oid) = current {
                head_chain.insert(c_oid);
                current = repo.find_commit(c_oid).ok().and_then(|c| c.parent_id(0).ok());
            }
        }
    }

    // Pre-reserve column 0 for ALL head_chain members via pending_parents.
    // This ensures that when a non-HEAD branch tip (e.g. B0) processes its
    // parent (e.g. C1) and finds C1 already in pending_parents at col 0,
    // it emits ForkLeft(branch_col -> 0) — the correct fork direction.
    // Without this, B0 would reserve C1 at B0's column, stealing it from HEAD.
    if !head_chain.is_empty() {
        // Ensure active_lanes has at least col 0
        active_lanes.push(None);
        for &hc_oid in &head_chain {
            pending_parents.insert(hc_oid, 0);
        }
    }
    ```

    **Why pre-populate works:** When B0 is visited first and processes its first parent C1, it finds C1 already in `pending_parents` at col 0 (pre-populated). It enters the "already pending" code path (lines 77-92) and emits `ForkLeft(B0_col -> 0)`. This is the correct fork direction: branch forks from main, not main from branch. When C2 is visited later, it finds itself in `pending_parents` at col 0 and gets col 0. C2's first parent C1 is also at col 0 in `pending_parents`, so it emits `Straight(0->0)`. The entire HEAD chain stays at col 0.

    **Change 2: Non-HEAD branch tips skip col 0 when finding a free lane.**

    Modify the column assignment block (lines 39-50). When a commit is NOT in `pending_parents` (new chain), it is necessarily NOT in `head_chain` (since all head_chain OIDs are pre-populated in `pending_parents`). So non-HEAD new chains must skip col 0:

    Replace the "New chain: find first free lane or create one" block (lines 43-49) with:

    ```rust
    // New chain — never a head_chain member (those are pre-populated in
    // pending_parents). Skip column 0 to keep it reserved for HEAD.
    let start_col = if !head_chain.is_empty() { 1 } else { 0 };
    if let Some(i) = active_lanes.iter().skip(start_col).position(|s| s.is_none()) {
        i + start_col
    } else {
        active_lanes.push(None);
        active_lanes.len() - 1
    }
    ```

    **Change 3: Re-occupy active_lanes when first-parent is already pending at same column.**

    In the first-parent handling block (the `if let Some(&existing_col) = pending_parents.get(&parent_oid)` branch, around lines 77-92), when `existing_col == col` (Straight continuation), the current code emits a Straight edge but leaves `active_lanes[col] = None` (it was consumed on line 69). This breaks pass-through edges for intermediate rows. Add a re-occupy after the edge push:

    ```rust
    if let Some(&existing_col) = pending_parents.get(&parent_oid) {
        // Parent already claimed — emit edge from col to existing column
        let edge_type = if existing_col == col {
            EdgeType::Straight
        } else if existing_col < col {
            EdgeType::ForkLeft
        } else {
            EdgeType::ForkRight
        };
        edges.push(GraphEdge {
            from_column: col,
            to_column: existing_col,
            edge_type,
            color_index: existing_col,
        });
        // If same column, re-occupy to maintain lane for pass-through edges
        if existing_col == col {
            if col < active_lanes.len() {
                active_lanes[col] = Some(parent_oid);
            }
        }
        // If different column, current col stays None (lane terminates here)
    }
    ```

    **Algorithm trace with all three changes applied (topology: C0<-C1<-C2(HEAD), C1<-B0(topic)):**

    Pre-loop: `pending_parents = {C2:0, C1:0, C0:0}`, `active_lanes = [None]`
    Revwalk order: B0, C2, C1, C0

    Row 0 — B0: In `pending_parents`? No. New chain, `start_col=1`, push to get col 1. `active_lanes = [None, None]`. Consume slot: `active_lanes[1] = None`. First parent C1: in `pending_parents` at col 0. `existing_col=0, col=1`. ForkLeft(1->0). `active_lanes[1]` stays None (lane ends). Result: B0 at col 1, edges=[ForkLeft(1->0)].

    Row 1 — C2: In `pending_parents` at col 0. `col=0`. `pending_parents.remove(C2)`. No pass-through (active_lanes has no occupied slots other than col 0). Consume: `active_lanes[0] = None`. First parent C1: in `pending_parents` at col 0. `existing_col=0, col=0`. Straight(0->0). Re-occupy: `active_lanes[0] = Some(C1)`. Result: C2 at col 0, edges=[Straight(0->0)].

    Row 2 — C1: In `pending_parents` at col 0. `col=0`. `pending_parents.remove(C1)`. No other active lanes for pass-through. Consume: `active_lanes[0] = None`. First parent C0: in `pending_parents` at col 0. `existing_col=0, col=0`. Straight(0->0). Re-occupy: `active_lanes[0] = Some(C0)`. Result: C1 at col 0, edges=[Straight(0->0)].

    Row 3 — C0: In `pending_parents` at col 0. `col=0`. `pending_parents.remove(C0)`. Consume: `active_lanes[0] = None`. No parents (root). Result: C0 at col 0, edges=[].

    Final: C2=col0, C1=col0, C0=col0, B0=col1. B0 has ForkLeft(1->0). HEAD chain has only Straight edges. All test assertions pass.

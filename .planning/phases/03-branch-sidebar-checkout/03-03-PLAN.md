    1. Add import: `import BranchSidebar from './components/BranchSidebar.svelte';`

    2. Add a `graphKey` counter and `handleRefresh` function. CommitGraph already reacts to `repoPath` via `$effect`, but incrementing a `{#key}` value forces a clean remount (re-triggers the effect with fresh state). This is the correct Svelte 5 pattern for triggering a controlled reload:
       ```typescript
       let graphKey = $state(0);
       function handleRefresh() { graphKey += 1; }
       ```

    3. Update the layout. Replace the current `<main>` block so that when a repo is open it renders BranchSidebar and CommitGraph side-by-side:
       ```svelte
       <main class="flex-1 overflow-hidden flex">
         <BranchSidebar repoPath={repoPath!} onrefreshed={handleRefresh} />
         {#key graphKey}
           <CommitGraph {repoPath} />
         {/key}
         <!-- Phase 4 adds StagingPanel here -->
       </main>
       ```
       Note: `repoPath!` non-null assertion is safe here because this branch only renders when `repoPath !== null`.

    4. Update `handleClose` to reset graphKey to avoid stale key state when user re-opens a different repo:
       ```typescript
       async function handleClose() {
         if (repoPath) {
           try { await safeInvoke('close_repo', { path: repoPath }); } catch {}
         }
         repoPath = null;
         repoName = '';
         graphKey = 0;
       }
       ```

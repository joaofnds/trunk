<objective>
Fix the intermittent collapsible-section click freeze in BranchSidebar.

The root cause (diagnosed in `.planning/debug/branch-sidebar-click-freeze.md`) is two interlocking bugs:

1. PRIMARY: The $effect sets `refs = null` synchronously before `loadRefs` resolves. This causes Remote, Tags, and Stashes BranchSection components to be destroyed (their `{#if}` guards become false), then recreated when data arrives. Clicks during the destroy/recreate window find no live handler.

2. SECONDARY: `loadRefs` has no cancellation guard. `handleCheckout` and `handleCreateBranch` both call it independently. Whichever resolves last wins, and a stale completion mid-session can trigger the PRIMARY destroy/recreate cycle again.

Purpose: Make collapsible sections stable and reliable every click, every time.
Output: Updated `BranchSidebar.svelte` with a loading boolean (never nulling refs) and a sequence counter on loadRefs.
</objective>

```svelte
async function loadRefs(path: string) {
  const seq = ++loadSeq;
  loading = true;
  try {
    const result = await safeInvoke<RefsResponse>('list_refs', { path });
    if (seq === loadSeq) {
      refs = result;
    }
  } catch {
    if (seq === loadSeq) {
      refs = null;
    }
  } finally {
    if (seq === loadSeq) {
      loading = false;
    }
  }
}
```

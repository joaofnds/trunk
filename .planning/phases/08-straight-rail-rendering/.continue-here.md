<decisions_made>

- Fork edges must be emitted on the PARENT commit's row, not the child's — this is the only way to make the visual connection originate from the parent dot
- Branch lanes stay active (active_lanes[col] = Some(parent_oid)) between child and parent so intermediate rows show pass-through rails, giving each branch its own visible lane
- is_branch_tip detection uses `active_lanes[col].is_none()` at column assignment time — correctly handles HEAD tip (pre-reserved but no child), new branches, and column reuse after merge
- Fork and Merge edges are visually similar but directionally opposite: Merge goes DOWN (toward secondary parent below), Fork goes UP (toward branch above)
- columns_seen approach for branch tip detection was rejected — breaks on column reuse
- pending_parents check for branch tip detection was rejected — HEAD chain pre-reservation makes HEAD tip appear as non-tip
</decisions_made>

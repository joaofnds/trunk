  <name>Task 2: Implement full lane rendering in LaneSvg.svelte</name>
  <files>src/components/LaneSvg.svelte</files>
  <action>
Rewrite `LaneSvg.svelte` to render three SVG layers in document order (later = on top in SVG):

**Layer 1 -- Vertical rail lines (bottom):**
- Filter `commit.edges` for `Straight` edges: `edge.from_column === edge.to_column`
- For each straight edge, render a `<line>` element:
  - `x1={cx(edge.from_column)}`, `x2={cx(edge.to_column)}`
  - `y1={-0.5}`, `y2={rowHeight + 0.5}` (0.5px overlap to eliminate sub-pixel gaps)
  - `stroke={laneColor(edge.color_index)}`
  - `stroke-width={2.5}`, `stroke-linecap="round"`
- This handles BOTH pass-through lanes AND the commit's own lane (backend emits both as Straight edges)

**Layer 2 -- Merge/Fork connection paths (middle):**
- Filter `commit.edges` for connection edges: `edge.from_column !== edge.to_column`
- For each connection edge, render a `<path>` with Manhattan routing:
  - Start at the commit dot center: `M cx(edge.from_column) cy`
  - Go horizontal toward target column: `H (cx(edge.to_column) -/+ cornerRadius)`
  - Arc 90 degrees: `A cornerRadius cornerRadius 0 0 sweepFlag cx(edge.to_column) (cy +/- cornerRadius)`
  - Go vertical to row edge: `V (rowHeight + 0.5)` for downward, `V -0.5` for upward
  - `fill="none"`, `stroke={laneColor(edge.color_index)}`, `stroke-width={2.5}`, `stroke-linecap="round"`
- Corner radius: `laneWidth / 2` (6px)
- Sweep flag logic for the SVG arc command:
  - MergeRight/ForkRight (to_column > from_column, going right then down): sweep=1
  - MergeLeft/ForkLeft (to_column < from_column, going left then down): sweep=0

**Build a `buildEdgePath(edge)` function** that handles all 4 edge types (MergeLeft, MergeRight, ForkLeft, ForkRight). The edge path construction:

For Merge edges (horizontal from commit, then vertical DOWN to row bottom):
```
M cx(from) cy
H cx(to) -/+ r       // horizontal toward target
A r r 0 0 sweep cx(to) cy+r  // 90-degree rounded corner
V rowHeight+0.5       // vertical to bottom of row
```

For Fork edges (horizontal from commit, then vertical UP to row top):
```
M cx(from) cy
H cx(to) -/+ r       // horizontal toward target
A r r 0 0 sweep cx(to) cy-r  // 90-degree rounded corner (upward)
V -0.5                // vertical to top of row
```

Sweep flag for upward corners is the OPPOSITE of downward:
- ForkRight (going right then up): sweep=0
- ForkLeft (going left then up): sweep=1

**Layer 3 -- Commit dot (top):**
- `<circle>` at `cx={cx(commit.column)}`, `cy={cy}`, `r={4}`
- `fill={laneColor(commit.color_index)}`

**Keep existing derived values:**
- `svgWidth`, `cx()`, `cy`, `laneColor()` helpers unchanged
- Add `cornerRadius = laneWidth / 2`
- Add `$derived` for `straightEdges` and `connectionEdges` filtered from `commit.edges`

**SVG element:** Keep `style="overflow: visible; flex-shrink: 0;"` (already set).

**Important per user decisions:**
- Horizontal edges draw ON TOP of vertical rails (SVG document order handles this -- paths after lines)
- Dot renders on top of everything (circle last in SVG)
- Rail is continuous behind dot (no gap, dot covers junction)
- All edges stay within their originating row
- No dashed lines (WIP connection is Phase 10)

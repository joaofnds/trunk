  <action>
Two issues to fix, both caused by the 6-column layout separating ref pills (Column 1) from the graph SVG (Column 2) into sibling divs.

**Fix A — Connector line (CommitRow.svelte):**

The connector line cannot live inside LaneSvg's SVG because the SVG is confined to the graph column div and cannot reach left into the ref column. Move the connector line out of LaneSvg.svelte and into CommitRow.svelte, where it can span across both columns using absolute positioning.

In CommitRow.svelte:
1. Make the outer row div `relative` (add `relative` to the class list).
2. After the ref column div and before (or after) the graph column div, add a connector line element that is absolutely positioned. It should render only when `commit.refs.length > 0 && commit.oid !== '__wip__'`. Use a simple `<div>` with absolute positioning:
   - `position: absolute`
   - `left: 0` (start at the left edge of the row, covering the ref column)
   - `right` calculated to end at the commit dot x-position within the graph column. The commit dot x = columnWidths.ref + (commit.column * 12 + 6) pixels from the row left edge (since laneWidth=12, cx = col*12 + 6). Set `width` to `columnWidths.ref + commit.column * 12 + 6` pixels.
   - `top: 50%`, `height: 1.5px`, `transform: translateY(-50%)`
   - `background: var(--lane-{commit.color_index % 8})`
   - `pointer-events: none` so it doesn't interfere with clicks
   - `z-index: 0` so it sits behind text but visible
3. Do NOT render the connector for WIP rows (`commit.oid === '__wip__'`).
4. The connector line in the ref column area should sit behind the ref pills. The RefPill div should have `position: relative; z-index: 1` so pills render on top of the line.

In LaneSvg.svelte:
5. Remove the entire connector line block (the `{#if commit.refs.length > 0 ...}` section, lines 82-92). The connector is now handled by CommitRow.

**Fix B — WIP dotted line (CommitRow.svelte):**

The WIP dotted line in LaneSvg.svelte draws below the SVG using `overflow: visible`, but the graph column div in CommitRow.svelte has `overflow-hidden` which clips it.

In CommitRow.svelte:
1. On the graph column div (Column 2), remove `overflow-hidden` from the class list. The graph column div currently has class `"flex items-center flex-shrink-0 overflow-hidden"`. Change to `"flex items-center flex-shrink-0"` (just remove `overflow-hidden`).
2. This allows the LaneSvg SVG's `overflow: visible` to work, letting the WIP dotted line extend into the next row visually.

No changes to the WIP line logic in LaneSvg.svelte — it is correct, just clipped.

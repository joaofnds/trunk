  <action>
Two changes in BranchSidebar.svelte:

**Fix 1 — Eliminate double refresh (white flash):**
Remove the `onrefreshed?.()` call from these four stash handler functions:
- `handleStashSave` (line 150): delete `onrefreshed?.();`
- `handleStashPop` (line 181): delete `onrefreshed?.();`
- `handleStashApply` (line 193): delete `onrefreshed?.();`
- `handleStashDrop` (line 211): delete `onrefreshed?.();`

Keep `await loadRefs(repoPath)` in each handler — this refreshes the sidebar's own ref list. The App.svelte repo-changed listener (lines 156-177) already handles the global refresh (refreshSignal, dirtyCounts, file diff refetch) with a 200ms debounce, so the onrefreshed call was causing a redundant immediate refresh on top of the debounced one, resulting in a white flash.

Do NOT remove onrefreshed?.() from `handleCheckout` or `handleCreateBranch` — those are branch operations that do not emit repo-changed from the backend, so they still need the explicit refresh callback.

**Fix 2 — Auto-expand stash section on '+' click:**
Change the `oncreate` handler on the Stashes BranchSection (line 343) from:
```
oncreate={() => { showStashForm = !showStashForm; stashCreateError = null; stashName = ''; }}
```
to:
```
oncreate={() => { showStashForm = !showStashForm; stashCreateError = null; stashName = ''; stashesExpanded = true; }}
```

Adding `stashesExpanded = true` ensures that when the user clicks '+' on a collapsed stash section, the section expands to reveal the create form. BranchSection only renders children when `expanded` is true (line 61 of BranchSection.svelte), so without this the form is created but invisible.

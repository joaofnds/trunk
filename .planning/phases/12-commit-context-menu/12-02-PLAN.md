- Add prop `oncontextmenu?: (e: MouseEvent, commit: GraphCommit) => void` to Props interface

**3. Wire context menu in CommitGraph.svelte**:

Add a `showCommitContextMenu(e: MouseEvent, commit: GraphCommit)` async function:

```typescript
import { Menu, MenuItem, PredefinedMenuItem } from '@tauri-apps/api/menu';
import { writeText } from '@tauri-apps/plugin-clipboard-manager';
import { ask } from '@tauri-apps/plugin-dialog';
```

Build menu items:
- "Copy SHA" -> `writeText(commit.oid)`
- "Copy Message" -> `writeText(commit.summary)`
- Separator
- "Checkout Commit..." -> handleCheckoutCommit(commit)
- "Create Branch..." -> handleCreateBranch(commit)
- "Create Tag..." -> handleCreateTag(commit)
- Separator
- "Cherry-pick" -> handleCherryPick(commit), `enabled: !commit.is_merge`
- "Revert" -> handleRevert(commit), `enabled: !commit.is_merge`

**Action handlers** (all use safeInvoke, catch TrunkError, display error via message() or console.error):

- **handleCheckoutCommit**: Per CONTEXT.md locked decision: (1) show `ask()` confirmation: "Checkout this commit in detached HEAD mode? You won't be on any branch. Create a branch afterward to save your work." with OK/Cancel. (2) If confirmed, `safeInvoke('checkout_commit', { path: repoPath, oid: commit.oid })`. (3) If error code "dirty_workdir", show error message via `message()` from dialog plugin.

- **handleCreateBranch**: Show InputDialog with title "Create Branch", one field: { key: "name", label: "Branch name", required: true }. On submit: `safeInvoke('create_branch', { path: repoPath, name: values.name, fromOid: commit.oid })`. On error (dirty_workdir or duplicate), show message(). Per locked decision: name field starts empty, always checkout new branch (backend handles this).

- **handleCreateTag**: Show InputDialog with title "Create Tag", two fields: { key: "name", label: "Tag name", required: true }, { key: "message", label: "Message (optional)", multiline: true }. On submit: `safeInvoke('create_tag', { path: repoPath, oid: commit.oid, tagName: values.name, message: values.message || "" })`.

- **handleCherryPick**: `safeInvoke('cherry_pick', { path: repoPath, oid: commit.oid })`. On conflict_state error, show message about resolving conflicts.

- **handleRevert**: `safeInvoke('revert_commit', { path: repoPath, oid: commit.oid })`. On conflict_state error, show message about resolving conflicts.

**InputDialog state management**: Use a `$state` variable like `let dialogConfig = $state<{...} | null>(null)` in CommitGraph.svelte. Render `{#if dialogConfig}<InputDialog ... />{/if}` at the bottom of the template. Action handlers set dialogConfig, onsubmit/oncancel clear it.

**Pass oncontextmenu to CommitRow**: Where CommitRow is rendered in CommitGraph.svelte, pass `oncontextmenu={showCommitContextMenu}`.

Graph refresh happens automatically -- backend commands update CommitCache and emit "repo-changed", which the existing listener in CommitGraph already handles.

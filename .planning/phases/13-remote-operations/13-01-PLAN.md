    - git_push passes ["push", "--progress"]

  </behavior>
  <action>
1. Create `src-tauri/src/commands/remote.rs` with:

   a) `classify_git_error(stderr: &str) -> TrunkError` function that matches stderr patterns:
      - "authentication failed" | "permission denied" | "could not read from remote" | "host key verification failed" | "connection refused" -> code "auth_failure"
      - "non-fast-forward" | "fetch first" | "failed to push some refs" -> code "non_fast_forward"
      - "no upstream" | "has no upstream branch" -> code "no_upstream"
      - anything else -> code "remote_error"
      All comparisons case-insensitive (lowercase the input).

   b) `async fn run_git_remote(args: &[&str], cwd: &std::path::Path, app: &tauri::AppHandle, repo_path: &str) -> Result<(), TrunkError>` helper:
      - Spawns `tokio::process::Command::new("git")` with the given args
      - Sets `current_dir(cwd)`, `env("GIT_TERMINAL_PROMPT", "0")`, `env("GIT_SSH_COMMAND", "ssh -o BatchMode=yes")`
      - Pipes both stdout and stderr via `Stdio::piped()`
      - Stores child PID in RunningOp state (passed as parameter or accessed via app state)
      - Reads stderr lines via `tokio::io::BufReader::new(stderr).lines()`
      - For each line: split on `\r`, take the last non-empty segment, trim, emit `app.emit("remote-progress", json!({"path": repo_path, "line": display}))` if non-empty
      - Collects all stderr lines into a `Vec<String>` for error classification on failure
      - Waits for child exit status
      - Clears RunningOp (set to None) after completion (success or failure)
      - On failure: joins collected stderr, calls `classify_git_error`, returns Err
      - On success: returns Ok(())

   c) Three Tauri commands following the established pattern (async fn, State params, AppHandle):

      `git_fetch(path: String, state: State<RepoState>, cache: State<CommitCache>, running: State<RunningOp>, app: AppHandle) -> Result<(), String>`:
      - Check RunningOp -- if Some, return Err with code "op_in_progress"
      - Get path_buf from RepoState
      - Call `run_git_remote(&["fetch", "--all", "--progress"], path_buf, &app, &path)`
      - On success: rebuild graph via `spawn_blocking` + `walk_commits`, insert into CommitCache, emit `repo-changed`
      - On error: serialize TrunkError to string

      `git_pull(path: String, strategy: Option<String>, state: State<RepoState>, cache: State<CommitCache>, running: State<RunningOp>, app: AppHandle) -> Result<(), String>`:
      - Check RunningOp -- if Some, return Err with code "op_in_progress"
      - Build args based on strategy: None -> ["pull", "--progress"], Some("ff") -> ["pull", "--ff", "--progress"], Some("ff-only") -> ["pull", "--ff-only", "--progress"], Some("rebase") -> ["pull", "--rebase", "--progress"]
      - Call `run_git_remote` with computed args
      - On success: rebuild graph, update cache, emit `repo-changed`

      `git_push(path: String, state: State<RepoState>, cache: State<CommitCache>, running: State<RunningOp>, app: AppHandle) -> Result<(), String>`:
      - Check RunningOp -- if Some, return Err with code "op_in_progress"
      - Call `run_git_remote(&["push", "--progress"], path_buf, &app, &path)`
      - On success: rebuild graph, update cache, emit `repo-changed`

      `cancel_remote_op(running: State<RunningOp>) -> Result<(), String>`:
      - Lock RunningOp, take PID if present
      - If PID exists: `unsafe { libc::kill(pid as i32, libc::SIGTERM); }` and set RunningOp to None
      - If no PID: return Ok (no-op)

   d) NOTE: The `run_git_remote` function needs the `RunningOp` state to store/clear PID. Pass it as a parameter: `running: &std::sync::Mutex<Option<u32>>`. The Tauri commands extract `&running.0` and pass it in. This avoids needing `State` inside the helper.

   e) For graph rebuild after remote ops: use `spawn_blocking` to call `walk_commits` (same pattern as commit_actions.rs). The remote command is already async (tokio::process), so after it completes, spawn_blocking handles the sync git2 graph walk.

2. Add `pub mod remote;` to `src-tauri/src/commands/mod.rs`.

3. Register all four commands in `src-tauri/src/lib.rs` invoke_handler:

   - Test classify with combined stderr output containing both progress and error lines

<task type="auto">
  <name>Task 1: Fix redo stack race condition by moving clearRedoStack to user-initiated operations</name>
  <files>src/components/Toolbar.svelte, src/components/CommitForm.svelte, src/components/CommitGraph.svelte</files>
  <action>
**Root cause:** In Toolbar.svelte, the repo-changed listener (lines 42-44) checks `isUndoing` to decide whether to clear the redo stack. But `handleUndo` resets `isUndoing = false` in its `finally` block (line 62) BEFORE the async repo-changed events arrive. So by the time repo-changed fires, `isUndoing` is already false, and `clearRedoStack()` wipes the stack.

**Fix â€” Toolbar.svelte:**
Remove the `clearRedoStack()` call entirely from the repo-changed listener (delete lines 42-44: `if (!isUndoing && !isRedoing) { clearRedoStack(); }`). The repo-changed handler should ONLY call `checkUndoAvailable()`. Also remove the `isUndoing` and `isRedoing` state variables since they are no longer needed for guarding. Remove the `finally` blocks that reset them in `handleUndo` and `handleRedo`. Remove the `isRedoing = true` / `isRedoing = false` lines in `handleRedo`. Remove the `isUndoing = true` / `isUndoing = false` lines in `handleUndo`. Remove the `clearRedoStack` import if no longer used in this file.

**Fix â€” CommitForm.svelte:**
Import `clearRedoStack` from `'../lib/undo-redo.svelte.js'`. In `handleSubmit()`, add `clearRedoStack()` as the FIRST line of the function (before any validation), so any user-initiated commit or amend clears the redo stack.

**Fix â€” CommitGraph.svelte:**
Import `clearRedoStack` from `'../lib/undo-redo.svelte.js'`. Add `clearRedoStack()` before the `safeInvoke('cherry_pick', ...)` call (around line 150) and before the `safeInvoke('revert_commit', ...)` call (around line 159).

This approach is race-free because clearing happens synchronously at the START of user actions, not asynchronously in event handlers.

**Fix â€” CommitForm.svelte:**

This notifies App.svelte that the subject is now empty, which sets `wipSubject = ''`, which flows to CommitGraph as `wipMessage={wipSubject.trim() || 'WIP'}` â€” correctly showing "WIP".
